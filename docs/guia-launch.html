Fase 0 — Conseguir las 4 llaves una sola vez

Antes de tocar la terminal, juntá estas 4 cosas. Son lo único que ningún robot puede hacer por vos.

¿Y Twitter/X? No necesita ninguna llave. Usa el oráculo oficial de Flap (XGeneralVerifier), que ya está funcionando en Robinhood Chain. Cero configuración.

Fase A — Publicar los contratos terminal · ~10 min

Acá "deployás" (publicás para siempre) la factory: el contrato-fábrica que crea un cofre de fees por cada builder. Se publica la versión exacta que aprobó el auditor de Flap (el tag audited-v3).

Fase B — Publicar la web vercel · ~10 min

La web es la cara del producto: /create para lanzar coins y /claim para cobrar. Además esconde al attester: un mini-servidor que verifica GitHub y firma los vouchers automáticamente.

Fase C — Lanzar el token piloto desde la web · lo divertido

Con la infraestructura arriba, el launch en sí es usar tu propio producto: la página /create hace todo (mina el salt, arma los parámetros, llama al portal de Flap).

Glosario para humanos

Las 10 palabras que aparecen en esta guía
wallet
Tu "cuenta" en la blockchain: una dirección pública (0x…, como un alias de banco) + una private key (la contraseña maestra).
private key
La clave secreta que CONTROLA una wallet. Quien la tiene, es el dueño. Jamás se comparte ni se pega en páginas.
gas
La comisión que cobra la red por procesar una transacción. Se paga en el ETH de esa red.
deploy
Publicar un contrato en la blockchain. Es permanente: no hay "deshacer".
factory
Nuestro contrato-fábrica: crea un escrow (cofre) por cada builder al que alguien le lanza una coin.
escrow / vault
El cofre de UN builder: acumula su tajada de los fees de trading, y solo se abre probando la identidad.
attester
El notario: una wallet-oráculo que firma "este GitHub es de esta persona". Vive en el servidor de la web.
salt / vanity
Un número que se "mina" localmente para que la dirección del token termine en 7777 (requisito del portal de Flap). La página /create lo hace sola.
RPC
La URL por donde tu compu le habla a la blockchain (como el servidor de un juego).
Guardian
El multisig oficial de Flap (no nuestro): el único rol privilegiado, para emergencias. Auditado y aceptado por ellos.
